# QA Testing Portfolio

This repo contains my collection of practical QA testing work, including manual test cases, bug reports, API testing notes, exploratory testing sessions, and open-source contribution evidence.

## Focus Areas

- Manual testing
- UI testing
- API testing
- Test case design
- Bug reporting
- Defect reproduction
- Exploratory testing
- SQL/database validation
- Open-source QA contributions

## Contents

- `bug-reports/` — structured bug reports with reproduction steps, expected vs actual results, severity, and environment details.
- `test-cases/` — manual test cases for web application features and user flows.
- `api-testing/` — API testing notes, Postman/Bruno collections, request/response validation, and error handling checks.
- `exploratory-testing/` — exploratory testing charters, session notes, and observations.
- `test-summary-reports/` — short summaries of testing performed, coverage, findings, and recommendations.
- `open-source-contributions/` — records of open-source issues tested, reproduced, reported, or contributed to.

## Goal

The goal of this portfolio is to demonstrate practical QA skills through clear documentation, structured testing, defect investigation, and real-world issue reporting.
